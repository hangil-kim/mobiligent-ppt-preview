3. 솔루션 및 사업화 타당성

DRT(시내/시외), 통근버스, 콜밴, 택시 등 다중 모빌리티를 하나의 플랫폼에서 운영하는 통합 솔루션

3.1 플랫폼 구조

[서비스 레이어]
시내 DRT | 시외 DRT | 통근버스 | 콜밴 | 택시
              ↓
[AI 배차 엔진] Routomile (OSRM 기반) + 실시간 최적화
              ↓
MSA 백엔드 | 멀티테넌트 SaaS | 정책 기반 정산 | 보안/개인정보
              ↓
승객 앱 | 기사 앱 | 관제 시스템 | 결제/정산

3.2 핵심 기술력

1) 자체 AI 배차 엔진 (Routomile)

오픈소스 대비 30% 성능 향상 / 다중 승객 합승 최적화, 실시간 경로 재계산, DRT 특화 알고리즘

AI 배차 정확도 95%, 목표 도착시간 오차 ±5분

2) 멀티테넌트 SaaS 구조

하나의 플랫폼으로 다수 지역·고객사 동시 운영 / 테넌트별 독립 설정(구역·요금·운영규칙) / 화이트라벨 확장

3) 데이터 주권 보장

운영 데이터 고객사 완전 소유 / 클라우드·온프레미스 선택 배포 / 개인정보보호법 준수 기본 탑재

4) 정책 기반 자동 정산

Rule Engine 기반 D+3 → D+1 익일정산 단축 / 운송사별 차등 정산 정책·다자간 수익 배분 자동화

3.3 사업화 타당성

항목모빌리전트의 사업화 강점
도입 구조SaaS 라이선스 + 구축 + 운영수수료 방식으로 즉시 매출화 가능
고객 적합성지자체, 운수사, 기업 통근, 콜밴 등 다중 고객군에 적용 가능
검증 수준실제 운영 실적과 파트너십을 통해 이미 시장 검증 진행

4. 비즈니스 모델 및 실현 가능성

플랫폼 구축(초기 매출) → 운영 수수료(반복 매출) → 데이터 플랫폼(미래 매출) 3단계 수익 구조

4.1 3단계 수익 구조

1단계: 플랫폼 구축 수익 (초기 매출)

지자체·운송사 대상 SaaS 라이선스 + 커스터마이징 구축 / 건당 2~6억원 수준

멀티테넌트 구조 → 고객 추가 시 한계비용 감소, 수익성 개선

2단계: 운영 수수료 수익 (반복 매출)

차량당 월정 이용료 5~10만원 + 거래액 기반 수수료 (운임의 5~10%)

고객사 사업 성장과 매출 성장 연동 → 장기 파트너십 유도, 이탈률 최소화

3단계: 데이터 플랫폼 수익 (미래 매출)

이동 패턴 분석 API → 지자체·연구기관 제공

복지·헬스케어 연계 데이터 서비스 (2028년 이후 본격화)
