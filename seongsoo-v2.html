3.2 핵심 기술력

1) 자체 AI 배차 엔진 (Routomile)

Routomile 자체 라우팅 엔진 — 오픈소스 대비 30% 성능 향상

다중 승객 합승 최적화, 실시간 경로 재계산, DRT 특화 수요응답형 배차 알고리즘

AI 배차 정확도 95% (±5분) / 외부 API 의존 없는 자체 엔진 → 비용 절감 + 커스터마이징 자유

2) 멀티테넌트 SaaS 아키텍처

하나의 플랫폼으로 다수 지역·고객사 동시 운영

테넌트별 독립 설정(구역·요금·운영규칙) / 신규 지역 오픈 시 설정만으로 확장 / 화이트라벨 구조

3) 데이터 주권 보장

운영 데이터(운행기록·배차로그·승객정보) 고객사 완전 소유

클라우드/온프레미스 선택 배포 / 안심번호·위치암호화·개인정보보호법 준수 기본 탑재

4) 정책 기반 자동 정산

Rule Engine 기반 → D+3 정산을 D+1 익일정산으로 단축

운송사별·서비스별 차등 정산 정책 → 다자간 수익 배분 자동화

3.3 범용 확장 구조

서비스 지역·차량 종류·승객 유형 제한 없는 범용 설계

5. 비즈니스 모델

플랫폼 구축(초기 매출) → 운영 수수료(반복 매출) → 데이터 플랫폼(미래 매출) 3단계 수익 구조

5.1 3단계 수익 구조

1단계: 플랫폼 구축 수익 (초기 매출)

지자체·운송사 대상 SaaS 라이선스 + 커스터마이징 구축

건당 2~6억원 (플랫폼 라이선스 + 사업지 오픈 지원 + 관리자 교육)

멀티테넌트 구조 → 고객 추가 시 한계비용 감소, 수익성 개선

2단계: 운영 수수료 (반복 매출)

차량당 월정 이용료 5~10만원 + 거래액 기반 수수료 (승객 운임의 5~10%)

무상 유지보수 이후 연간 기술지원료 전환

고객사 사업 성장과 매출 성장 연동 → 장기 파트너십 유도, 이탈률 최소화

3단계: 데이터 플랫폼 (미래 매출)

이동 패턴 분석 API → 지자체·연구기관 제공

의료 통원·시니어 이동 패턴 등 복지·헬스케어 연계 데이터 서비스 (2028년 이후 본격화)

5.2 고객 유형별 수익 구조

고객 유형결제 방식이용 규모현재 상태
지자체구축비 + 월정 운영료지역당 차량 20대+셔클 DRT 7개+ 지역 운영, 자체 DRT 오픈 추진
민간 운수사업자구축비 + 기술지원료대형 운송 그룹 5,500대+KD운송그룹(Nova) 계약, 전국콜밴조합 MOU
기업 (통근)구축비 + 대당 월정료통근 수요 대기업제안 진행 중
모빌리티 플랫폼사운영관리 수수료플랫폼 운영 위탁현대자동차 셔클 운영관리
